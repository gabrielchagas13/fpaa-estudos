📋 Objetivo

Verificar se os problemas abaixo pertencem à classe P, NP e/ou co-NP. Forneça o problema de decisão correspondente (certificado positivo e negativo). Explique seu raciocínio.

Exercícios
Exercício 1 — Maior valor de um vetor
Encontrar o maior valor de um vetor.
Problema de decisão: "Dado vetor V de n elementos e valor k, o maior valor de V é ≥ k?"

Certificado +: Índice i com V[i] ≥ k e V[i] = max. Verificação: percorrer o vetor confirmando → O(n).

Certificado −: Percorrer e verificar que todos < k → O(n).

Resolução: Percorrer o vetor uma vez → O(n) → polinomial.

Classificação: ∈ P
Exercício 2 — Avaliar se um número é primo
Avaliar se um dado número é primo.
Problema de decisão: "Dado n, n é primo?"

Certificado +: Verificar primalidade (teste de divisibilidade até √n) → O(√n).

Certificado −: Fornecer um divisor d de n onde 1 < d < n → verificar em O(1).

Resolução: Algoritmo AKS (2002) prova primalidade em tempo polinomial O((log n)^6).

Classificação: ∈ P (desde AKS 2002; antes era ∈ NP ∩ co-NP)
Exercício 3 — Ordenar um vetor em ordem crescente
Ordenar um vetor de maneira crescente.
Problema de decisão: "Dado vetor V, existe permutação de V que está ordenada em ordem crescente?"

Nota: Este é um problema de otimização, não decisão pura. Sua versão de decisão é trivial (resposta sempre SIM para qualquer vetor).

Resolução:** Qualquer algoritmo de ordenação: Merge Sort, Heap Sort em O(n log n) → polinomial.

Classificação: ∈ P
Exercício 4 — Avaliar se um grafo é hamiltoniano
Avaliar se um grafo é hamiltoniano.
Problema de decisão: "Dado grafo G, existe ciclo que visita todos os vértices exatamente uma vez?"

Certificado +: Fornecer a sequência de vértices do ciclo Hamiltoniano → verificar que é ciclo simples visitando todos → O(V²).

Certificado −: Nenhum conhecido em tempo polinomial.

Status: Nenhum algoritmo polinomial conhecido para resolvê-lo. Brute force: O((V-1)!) → exponencial.

Classificação: ∈ NP-Completo (∈ NP e NP-Difícil; não se sabe se está em P)
Exercício 5 — Avaliar se um grafo é euleriano
Avaliar se um grafo é euleriano.
Problema de decisão: "Dado grafo G conexo, existe ciclo que visita todas as arestas exatamente uma vez?"

Teorema de Euler: Um grafo conexo tem ciclo Euleriano ⟺ todo vértice tem grau par.

Resolução: Verificar grau de cada vértice → O(V) → polinomial.

Certificado +: Verificar que todos os vértices têm grau par → O(V).

Classificação: ∈ P (é fácil!)
Exercício 6 — Encontrar uma árvore geradora de um grafo
Encontrar uma árvore geradora de um grafo.
Problema de decisão: "Dado grafo G, existe uma árvore geradora (subgrafo conexo acíclico com V-1 arestas)?"

Resolução: BFS/DFS → O(V+E) → polinomial. Se o grafo é conexo, sempre tem árvore geradora.

Certificado +: Fornecer o conjunto de V-1 arestas que forma a árvore → verificar conexidade e aciclicidade → O(V+E).

Classificação: ∈ P
Exercício 7 — Verificar se um grafo é 3-colorável
Verificar se um grafo é 3-colorável.
Problema de decisão: "Dado grafo G, existe atribuição de cores {1,2,3} aos vértices tal que nenhuma aresta conecta vértices da mesma cor?"

Certificado +: Fornecer a coloração → verificar que nenhuma aresta conecta vértices com mesma cor → O(E).

Certificado −: Nenhum conhecido em tempo polinomial.

Status: 3-SAT reduz para 3-Coloração. Nenhum algoritmo polinomial conhecido.

Classificação: ∈ NP-Completo
Exercício 8 — Verificar se um grafo é 2-colorável
Verificar se um grafo é 2-colorável.
Problema de decisão: "Dado grafo G, G é bipartido (2-colorável)?"

Resolução: BFS/DFS tentando 2-colorir → O(V+E) → polinomial. Se houver ciclo ímpar, não é bipartido.

Certificado +: Fornecer a bipartição (dois conjuntos de vértices) → verificar que não há aresta dentro dos conjuntos → O(E).

Classificação: ∈ P (contraste com 3-coloração!)
Exercício 9 — Encontrar o maior subgrafo completo (Clique)
Encontrar o maior subgrafo completo (i.e., clique) de um grafo.
Problema de decisão: "Dado grafo G e inteiro k, existe clique de tamanho ≥ k?"

Certificado +: Fornecer k vértices que formam clique → verificar que todos são mutuamente adjacentes → O(k²).

Certificado −: Nenhum conhecido em tempo polinomial.

Status: Brute force: testar todas as combinações de k vértices → C(V,k) = exponencial.

Classificação: ∈ NP-Completo
Exercício 10 — Encontrar o conjunto independente máximo
Encontrar o conjunto independente máximo de um grafo.
Problema de decisão: "Dado grafo G e inteiro k, existe conjunto independente de tamanho ≥ k?"

Certificado +: Fornecer k vértices sem nenhuma aresta entre eles → verificar ausência de arestas → O(k²).

Certificado −: Nenhum conhecido em tempo polinomial.

Relação com Clique: Conjunto independente em G ⟺ Clique no complemento de G.

Classificação: ∈ NP-Completo
Exercício 11 — Encontrar a cobertura mínima de vértices
Encontrar a cobertura mínima de vértices de um grafo.
Problema de decisão: "Dado grafo G e inteiro k, existe conjunto de ≤ k vértices que toca todas as arestas?"

Certificado +: Fornecer k vértices → verificar que toda aresta tem pelo menos um extremo nesse conjunto → O(k·E).

Certificado −: Nenhum conhecido em tempo polinomial.

Relação com Independente: S é cobertura de vértices ⟺ V \ S é conjunto independente.

Classificação: ∈ NP-Completo
Exercício 12 — Encontrar a cobertura mínima de arestas
Encontrar a cobertura mínima de arestas de um grafo.
Problema de decisão: "Dado grafo G e inteiro k, existe conjunto de ≤ k arestas que toca todos os vértices?"

Resolução: Este é equivalente ao problema de emparelhamento máximo, que pode ser resolvido em tempo polinomial via algoritmos de fluxo (Hopcroft-Karp: O(E√V)) ou programação linear.

Certificado +: Fornecer k arestas → verificar que todo vértice tem pelo menos uma aresta nesse conjunto → O(k + V).

Classificação: ∈ P (contraste com cobertura de vértices!)
Exercício 13 — Contar a quantidade de subconjuntos em um conjunto
Contar a quantidade de subconjuntos existentes em um conjunto qualquer.
Problema de decisão: "Dado conjunto S de n elementos e inteiro k, existe mais de k subconjuntos de S?"

Resolução: Um conjunto de n elementos tem exatamente 2^n subconjuntos. Isso é uma fórmula fixa → O(1) para calcular (2^n é exponencial em valor, não em representação em bits).

Resosta: Sempre SIM, pois 2^n > k para qualquer k ≤ 2^(n-1). Se k > 2^n, resposta é NÃO.

Certificado +/−: Comparação aritmética: 2^n ≥ k? → O(1).

Classificação: ∈ P
