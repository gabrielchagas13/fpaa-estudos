Fundamentos de Projeto e Análise de Algoritmos

↑ Voltar ao Início ← Voltar ao Simulado Ir para P, NP e co-NP → Ir para Prova Antiga →
Ideia geral: você tem um problema A pra resolver, mas só tem o algoritmo de B disponível. Você transforma A em B, roda B, e lê a resposta de volta. A transformação precisa ser polinomial e preservar a resposta (SIM↔SIM, NÃO↔NÃO).
Exercício 1 — Caminho com mais arestas (Dijkstra / Bellman-Ford)
Maior quantidade de arestas entre dois vértices
Usando Dijkstra (supondo que aceita pesos negativos) para resolver
Problema
Dado grafo não ponderado e dois vértices s e t, achar o caminho com mais arestas entre eles.
Algoritmo disponível
Dijkstra → acha o caminho de menor peso. Na prática usamos Bellman-Ford pois aceita pesos negativos.
Transformação: Atribua peso w'(u,v) = -1 a toda aresta. O caminho com mais arestas fica com o menor custo total (mais negativo).

Etapas:
  1. Crie G' com os mesmos vértices e arestas de G
  2. Atribua w'(u,v) = -1 a toda aresta
  3. Rode Bellman-Ford(G', s, t)
  4. Custo retornado = -k → caminho tem k arestas
Por que funciona: Cada aresta adiciona -1 ao custo. Mais arestas = custo mais negativo = mínimo no Bellman-Ford.
Tempo: O(E) construção + O(VE) Bellman-Ford → polinomial ✅
1 / 4 grafo original
Exercício 2 — Emparelhamento máximo (Ford-Fulkerson)
Maior conjunto de arestas sem vértices em comum
Usando fluxo máximo para resolver
Problema
Dado grafo G, achar o maior conjunto de arestas sem vértices em comum (emparelhamento máximo).
Algoritmo disponível
Ford-Fulkerson → acha fluxo máximo numa rede.
Transformação: Duplicar cada vértice v em v_in e v_out com aresta interna de capacidade 1. Adicionar fonte s e sumidouro t.

Etapas:
  1. Duplique cada vértice v em v_in e v_out, ligados por aresta v_in → v_out com capacidade 1
  2. Crie fonte s → v_in para todo v (capacidade 1)
  3. Crie sumidouro v_out → t para todo v (capacidade 1)
  4. Para cada aresta (u,v) do grafo: crie u_out → v_in e v_out → u_in (capacidade 1)
  5. Fluxo máximo = tamanho do emparelhamento
Por que funciona: Cap 1 na aresta interna garante que cada vértice aparece no máximo 1 vez no emparelhamento.
Tempo: O(V + E) construção → polinomial ✅
1 / 4 grafo original
Exercício 3 — Conjunto independente máximo (Cobertura mínima de vértices)
Maior conjunto de vértices sem nenhuma aresta entre eles
Usando cobertura mínima de vértices
Problema
Dado grafo G, achar o maior conjunto de vértices sem nenhuma aresta entre eles.
Algoritmo disponível
CoberturaMinima → acha o menor conjunto de vértices que toca todas as arestas.
Relação fundamental: S é independente ⟺ V \ S é cobertura

Etapas:
  1. Rode CoberturaMinima(G) → obtém C*
  2. Conjunto independente máximo: I* = V \ C*
  3. Tamanho: |I*| = |V| - |C*|
Por que funciona: O que não faz parte da cobertura não pode ter aresta — se tivesse, a cobertura não estaria cobrindo ela. Logo V \ C* é independente. Menor cobertura = maior independente.
Tempo: O(V) para complementar → polinomial ✅
1 / 3 grafo original
Exercício 4 — Maior clique (Conjunto independente máximo)
Maior subgrafo completo de um grafo
Usando conjunto independente máximo via grafo complementar
Problema
Dado grafo G, achar o maior conjunto de vértices onde todos são conectados entre si (clique máxima).
Algoritmo disponível
ConjuntoIndependenteMaximo → acha maior grupo sem arestas.
Relação fundamental: S é clique em G ⟺ S é independente em Ḡ

Etapas:
  1. Construa Ḡ (complemento de G): arestas que existem em G somem, as que não existiam aparecem
  2. Rode ConjuntoIndependenteMaximo(Ḡ) → obtém I*
  3. I* é a maior clique de G
Por que funciona: Se S é clique em G (todos pares conectados), nenhum par tem aresta em Ḡ → S é independente em Ḡ. O raciocínio inverso também vale.
Tempo: O(V²) para construir Ḡ → polinomial ✅
1 / 3 grafo original G
Exercício 5 — Verificar se grafo é bipartido (Coloração mínima)
Verificar se vértices podem ser divididos em dois grupos sem arestas internas
Usando coloração mínima de vértices
Problema
Dado grafo G, verificar se os vértices podem ser divididos em dois grupos onde toda aresta vai de um grupo para o outro.
Algoritmo disponível
ColoracaoMinima → acha o menor número de cores χ(G) para colorir vértices sem adjacentes iguais.
Relação fundamental: G é bipartido ⟺ χ(G) ≤ 2

Etapas:
  1. Rode ColoracaoMinima(G) → obtém χ*
  2. Se χ* ≤ 2 → G é bipartido ✅
  3. Se χ* ≥ 3 → G não é bipartido ✗
Por que funciona: Bipartido = cada lado recebe uma cor. Se 2 cores bastam (χ ≤ 2), pode separar em dois grupos. Se precisa de 3+ cores, tem ciclo ímpar → não é bipartido.
Tempo: O(1) para comparar → polinomial ✅
1 / 3 grafo bipartido
